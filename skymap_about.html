About νetal Skymap

This tool seeks to simplify and make accessible the emerging branch of astrophysics called multi-messenger astronomy. Those pioneering this field are working to combine observations of ‘messenger particles’ - neutrinos, cosmic rays, electromagnetic radiation and gravitational waves - in order to gain an enhanced understanding of their sources. The recent confirmation of both the existence and detectability of gravitational waves has helped to propel the field to the forefront of the physics community, and there is much anticipation as to what it might hold for the future.

With this in mind, it will become increasingly important to make sure that multi-messenger astronomy does not remain confined only to those at the cutting edge of research and academia. The Multi-Messenger Skymap is designed to be an easy-to-use tool for visualizing the core concepts of multi-messenger astronomy, as well as conducting low level data analysis by eye. Implemented by default are data releases from the IceCube neutrino detector in Antarctica, NASA’s gamma-ray LAT (Large Area Telescope) aboard the Fermi satellite, and the High-Altitude Water Cherenkov Gamma-Ray Observatory (HAWC) in Mexico. Data from these detectors, as well as from any of the user’s choosing, can also be uploaded and displayed on the skymap.

This map was created by five high-school students during a six week internship at WIPAC in the summer of 2017. Special thanks goes to Joshua Wood, David Schultz, Sílvia Bravo and Amanda Nothem for their advice, support, and guidance during the project.

How to use this tool



Although not necessary, you may choose to upload your own files rather than use the ones we provide. In this case, find the button in the upper left corner labeled "choose files", and when clicked, will let you choose a file from your computer. However, this site will only be able to display data from files in a certain format, and this format will be further explained in the section below called "File Conversion". Once uploaded, you will have the option of specifying the file as "neutrino", "source", "gamma ray", or "other". Doing so will place the file in one of the following four sections:

  • Iceube

    • This selection allows the user to display neutrino data from IceCube.

    First, select which file you want to pull data from. If you would like to select more than one file at a time, press and hold command or control and select all the files you wish to display.

    Next, since neutrinos are labeled as either tracks or showers (based on angle error), select which you want to show.

    Lastly, the slider will allow you to choose a range of energy to display. The maximum and minimum values of this energy slider are automatically set to the maximum and minimum of the files selected. The boxes with colors, when clicked, will allow you to choose the gradient of color on which the energy is displayed, specifying which color is to be associated with the maximum and which with the minimum. Also, beneath this slider, you will find a checkbox option which will allow you to switch the range of energy between a logarithmic and a linear scale.

  • Fermi/Gamma Ray Events

    • Nothing new here. This section has a selection of files to choose from and the energy slider below operates in the same way.

  • Gamma Ray Sources

    • Similarly to the last two, this also has a selection of files to choose from. However, unlike the last two, the data to be manipulated by the user is different, and thus the options are as well.

    First, and fairly self-explanatory, is the array of checkboxes allowing you to choose which source types are displayed, out of the following: unidentified, composite SNR, shell, pulsar wind nebula, HBL, massive star cluster, and SNR molecule cloud.

    Next, the user may choose between displaying the color of the data according to spectral index, flux, or neither. The energy sliders and the color work the same as they did previously, only they are specifically for spectral index or flux, and the linear/logarithmic option exists here as well.

  • Other Data

    • Unlike the previous three options, this one has to do with data you upload yourself and do not specify which section it should fall under (otherwise, if you do specify, the name of your file should appear in the file selections of the other sections).

    Having selected a file, the name of this file should have been added to the area which in previous sections had selectable files. Similarly to previous sections, you can choose any or several of the files you upload.

Having selected which files from which telescopes you want to display, find the options at the top of the page (IceCube, Fermi Events, Gamma Ray Sources, Other), select which you would like displayed and press submit to run the process. Only press submit if all of the options you want to be displayed are selected, including any options described below.





In the upper right-hand corner of the page will be an option between two checkboxes labeled "Galactic Coordinates" and "Equatorial Coordinates". The purpose of this option should be fairly self explanatory. If Galactic Coordinates is selected, the data will appear on the SkyMap in Galactic Coordinates, the equator being the galactic plane. If Equatorial Coordinates is selected, the data will be displayed on the SkyMap in equatorial coordinates, drawing the galactic plane on the map.





The last two options which you will see are labeled as "Toggle filtering by time" and "Toggle zoom and pan".

Toggle filtering by time

When this option is selected, you will be presented with several more options in order to specify which data is shown: year, month, day, hour, minute, second, and margin. These options will allow you to choose which time period will be displayed. The "margin" option will allow you to choose the range of time above and below what you selected. The default, 31536000 seconds, is one year.

Toggle zoom and pan

This option, when selected, will allow you to zoom and pan on the SkyMap. Zooming can be accomplished by scrolling while hovering over the map. Panning can be accomplished by clicking and dragging the map.





In order to understand the file conversion, one must understand the format of the files which this site reads. First, the text file (it must be a text file) must have, as its first line, a header, which must have at least these two headers: dec and ra (for declination and right ascension). Other headers, if they are values included in the options in our site, will be read as well, however are not necessary. If any headers which are more than one word long (anything separated by a space), the file must be edited (ex: Neutrino En. will become NeutrinoEn.). Anything inside parentheses or brackets will not be read by the program in both the header and the following data.

If the file you want to convert is a .FITS file, the button in the upper left of the screen will give you information on how to convert that file. Otherwise, if the file is in a .txt format, the site itself will convert it for you, as long as the file follows the parameters already described. An acceptable file header would be:

err RA Dec Distributed_Energy (MeV) topology
